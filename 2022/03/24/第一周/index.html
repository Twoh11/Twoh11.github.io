单例模式饿汉式1

单例模式懒汉式2

饿汉式就是你没有调用有可能就已经创建了，在类的加载过程中就会被调用，例如new一个对象，构造子类对象父类，调用一个静态属性或者方法时这三种都会引起类的加载。但是懒汉就是我需要的时候再创建这个实例。

Java底层有一个runtime类就是用到了单例设计模式（饿汉式）

接口的使用细节

1）接口不能被实例化

2）接口中所有的方法是public方法，接口中抽象方法，可以不用abstract修饰

void aaa() 实际上 abstract void aaa();

3)一个普通类实现接口，就必须将该接口的所有方法实现

5)一个类可以实现多个接口

6）接口中的属性，只能是final的，而且是public static fuinal修饰符。比如 int a = 1; 实际上是 public static final int a = 1;(必须初始化）

7)接口中属性的访问形式：接口名.属性名

8）一个接口不能继承其它的类，但是可以继承多个别的接口

9）接口的修饰符，只能是public或默认
